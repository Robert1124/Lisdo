Lisdo 的清晰节奏：先草稿，再保存

Lisdo 用一个大面板讲清核心动作：不是直接生成待办，而是把捕获内容先变成可检查的 AI 草稿。

走一遍流程
01

Capture

文字、截图、照片、语音和 Mac 捕获都进入同一个 inbox。

02

Draft

Vision OCR 和 BYOK provider 生成严格 JSON 草稿。

03

Review

用户编辑标题、摘要、清单、日期和分类。

04

Save

确认后保存为 category todo，并通过 iCloud 同步。

安静一点，但工作流要完整

Lisdo 的动效轻盈、克制、容易理解。视觉回到原生 Apple 产品：纯净表面、近黑文字、低饱和状态色和克制阴影。

页面避免夸张 AI 营销语言，因为 Lisdo 的价值不是炫技，而是让 messy capture 变成可检查、可同步、可执行的任务。

常见问题

Mac 端已经开源，现在可以自己 build 使用；但还未完成完整测试和 debug，所以暂时没有安装包。测试完成后会 release 安装包。iOS 版本也在同步开发，完成后会上传 TestFlight。

不会。Lisdo 的产品流要求所有 AI 输出先成为 draft。用户 review、编辑并确认后，才保存为正式 todo。

MVP 1 设计为 SwiftData + CloudKit，同步 categories、captures、drafts、todos 和 pending queue metadata。API keys 和 provider secrets 只保存在本机 Keychain。

默认策略是本机 Vision OCR，并同步 OCR text 与 metadata；原图默认不通过 iCloud 同步。

Mac 菜单栏捕获目前可用。语音捕获暂不可用，会在核心截图/粘贴转 todo 流程稳定后继续开发。
